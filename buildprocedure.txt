if there shall be some manipulation with the charmm-gui generated psf & pdbb files with vmd or pymol then the new pdb file shall be converted to CRD format so it can be read y CHARMM-GUI FF CONVERTOR! to do so use pdb2crd.tcl file in github with following script in vmd:

(vmd -dispdev text)
mol new solute.psf 
mol addfile solute.pdb
source pdb2crd.tcl
writecharmmcoor "solute.crd" 0 "normal"

Also if a new psf is generated by vmd (e.g because new CAL ions are added), the new psf file shall be converted to XPLOR format so it can be read by CHARMM-GUI FF. use following script in vmd:

package require psfgen
readpsf solute.psf
coordpdb solute.pdb
writepsf x-plor solute-XPLOR.psf

Add "XPLOR" keyword to the header of the Xplor formatted psf file, solute-XPLOR.psf as shown below:
change "PSF EXT CMAP" to "PSF EXT CMAP XPLOR"

gmx editconf -f solute.gro -o boxed.gro -center 22.5 12.5 10 -box 45 15 20 

gmx solvate -cp boxed.gro -cs spc216.gro -o boxedsol.gro -p topol.top

add below line to topol.top under other #Include forcefield parameters
#include "toppar/tip3p.itp"

copy tip3p.itp to toppar folder

if solute topology is generated via charmm-gui FFconvertor with no solvent, add below lines to the end of [atomtypes] section in forcefield.itp in toppar folder for tip3 solvent LJ parameters:
      HT     1     1.0080      0.417     A    4.00013524445e-02    1.924640e-01 
      OT     8    15.9994     -0.834     A    3.15057422683e-01    6.363864e-01
	  
gmx grompp -f ions.mdp -c boxedsol.gro -p topol.top -o ions.tpr
gmx genion -s ions.tpr -o boxedsoln.gro -p topol.top -pname NA -nname CL -neutral -conc 0.15  

add  below lines to topol.top under other #included parameters:
#include "toppar/NA.itp"
#include "toppar/CL.itp"
copy "NA.itp" & "CL.itp" to toppar folder

add following lines to the end of forcefield.itp:
#include "charmm36-CL.itp"
#include "charmm36-NA.itp"
copy "charmm36-NA.itp" & "charmm36-CL.itp" files to toppar folder
